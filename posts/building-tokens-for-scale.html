Building Tokens for Scale

June 2, 2023

Design tokens are the foundation of any robust design system. They're the atoms from which all other components are built, providing a single source of truth for your design language. In this post, we'll explore how to build a token system that scales effectively across products and platforms.

What Are Design Tokens?

Design tokens are named entities that store visual design attributes. They're essentially variables that can be used across different platforms and codebases. Instead of hardcoding values like #FF0000 or 16px, you define tokens like --color-primary or --spacing-medium that can be referenced throughout your system.

Common categories of design tokens include:

  • Colors: Primary, secondary, accent, text, background, etc.
  • Typography: Font families, sizes, weights, line heights
  • Spacing: Margins, paddings, gaps
  • Borders: Widths, radii, colors
  • Shadows: Box shadows, drop shadows
  • Animation: Durations, easing functions
  • Breakpoints: Screen size thresholds

The Token Hierarchy

A well-structured token system typically has multiple layers of abstraction:

1. Global Tokens

These are your most basic values, often representing the raw palette:

--color-blue-500: #0066FF;
--color-gray-100: #F5F5F5;
--spacing-4: 4px;
--spacing-8: 8px;

2. Semantic Tokens

These map global tokens to their functional use:

--color-primary: var(--color-blue-500);
--color-background: var(--color-gray-100);
--space-small: var(--spacing-4);
--space-medium: var(--spacing-8);

3. Component-Specific Tokens

These apply semantic tokens to specific components:

--button-primary-background: var(--color-primary);
--button-padding: var(--space-medium);

This layered approach allows for flexibility and consistency. If your brand color changes, you only need to update the semantic token mapping, and all components using that token will update automatically.

Implementing Tokens Across Platforms

One of the challenges of design tokens is maintaining consistency across different platforms and technologies. Here are some approaches:

CSS Custom Properties

For web applications, CSS variables are a natural fit:

:root {
  --color-primary: #0066FF;
  --spacing-medium: 8px;
}

.button {
  background-color: var(--color-primary);
  padding: var(--spacing-medium);
}

JSON/YAML as a Source of Truth

Many teams store tokens in a platform-agnostic format like JSON or YAML:

{
  "color": {
    "primary": {
      "value": "#0066FF"
    }
  },
  "spacing": {
    "medium": {
      "value": "8px"
    }
  }
}

This can then be transformed into platform-specific formats using tools like Style Dictionary or Tokens Studio.

Native Platform Variables

For native applications, tokens can be implemented using platform-specific variables:

  • Swift: Using color assets and dimension resources
  • Android: Using resource files (colors.xml, dimens.xml)
  • React Native: Using a theme provider

Best Practices for Scaling Tokens

1. Use a Consistent Naming Convention

Adopt a clear, predictable naming pattern. Common approaches include:

  • Category-Property-Scale: --color-background-primary
  • Purpose-Property-Variant: --button-background-hover

2. Document Your Token System

Comprehensive documentation helps teams understand how and when to use specific tokens. Include:

  • Visual examples of each token
  • Usage guidelines and constraints
  • Implementation details for different platforms

3. Plan for Theming

Design your token system with theming in mind from the start. This might include:

  • Light and dark mode variants
  • Brand-specific themes for different products
  • Seasonal or campaign-based themes

4. Version Your Tokens

As your design system evolves, so will your tokens. Implement versioning to manage changes without breaking existing implementations.

Tools for Managing Design Tokens

Several tools can help you manage and distribute design tokens:

  • Style Dictionary: An open-source tool that transforms design tokens into platform-specific variables
  • Theo: Salesforce's token management system
  • Tokens Studio: A Figma plugin for creating and managing tokens
  • Specify: A tool for syncing design tokens between design tools and code

Conclusion

Design tokens are the foundation of a scalable design system. By carefully structuring your tokens with clear naming conventions, appropriate levels of abstraction, and cross-platform compatibility in mind, you can create a flexible system that grows with your organization.

Remember that building a token system is not a one-time effort—it's an ongoing process that requires maintenance and refinement as your products and design language evolve. But the investment pays off in consistency, efficiency, and the ability to make sweeping changes with minimal effort.
